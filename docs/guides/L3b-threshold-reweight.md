# L3b — Threshold reweight (перевес к цели)

> Накопительно: всё из [L1](L1-static-in-kind.md)+[L2](L2-readonly-nav.md) + движок ребаланса (ветка «веса»). Термины — [README.md](README.md).

## 1. Что нового

Поддерживаем **целевые веса**: equal-weight (каждый по 1/N) или фиксированные доли. Когда веса уезжают от цели (дрейф), ребалансим назад — по календарю или при пробое порога. Фиат-аналоги: **RSP** (equal-weight S&P, десятки $B, Invesco 2026; ребаланс поквартально к 1/N) — это календарный случай; **threshold**-случай (ребаланс при пробое band) — vol-target / тактические фонды.

> Ключевое отличие от L3a: тут меняем **веса существующих** активов, не состав. И **оракул становится load-bearing** — без цены нельзя измерить дрейф (вес `wᵢ = (qtyᵢ·priceᵢ) / NAV` — доля стоимости позиции в стоимости всей корзины; `qty·price` это стоимость позиции, а не сам вес).

## 2. Новые термины

| Термин | Простыми словами |
|---|---|
| **target weight (целевой вес)** | Желаемая доля актива по стоимости (напр. 1/N для equal-weight). |
| **drift (дрейф)** | Насколько текущие веса уехали от целевых из-за движения цен. |
| **threshold / band** | Порог: ребаланс, когда дрейф > X% (а не только по календарю). |
| **load-bearing оракул** | Цена теперь нужна для *решения* о ребалансе, не только для показа. |

## 3. Что меняется во входе/выходе

- **`create`/`redeem`:** как в L1 (in-kind, без цены).
- **Новая операция — `reweight` (ребаланс весов):** keeper-операция; читает NAV (из L2), считает дрейф, обменивает дельту in-kind к целевым весам.
- **Выход не паузится.**

## 4. Новый модуль и функции

**RebalanceEngine (ветка reweight):**
```
setTargetWeights(weight[])              // curator + timelock
checkDrift() → (drifted, deltas[])      // читает NAV, сравнивает с целью
reweight()                              // keeper: своп дельты к целевым весам
```
Триггер: `calendar` (по дате) ИЛИ `threshold` (дрейф > band). Общий с L3a исполнитель свопа (value-preserving, market-open).

## 5. Реализация по слоям

### Контракты
- `target-weight controller`, `drift-check` (читает `NAVEngine.latestNAV`), non-pro-rata swap.
- Инвариант: своп value-preserving в пределах `maxSlippageBps`, market-open + свежая цена.

### Бекенд
- **Drift-монитор** (постоянно считает отклонение от цели).
- **Trigger-evaluator** (календарь/порог; sustained-проверка, чтобы не дёргаться на шуме).
- **Rebalance scheduler / keeper.**

### Фронтенд
- Конфиг целевых весов, view текущего дрейфа, лог ребалансов.

## 6. Сквозной step-by-step — reweight по порогу

| # | Слой | Действие |
|---|---|---|
| 1 | BE | Drift-монитор: NVDA уехала на +7% от цели (band 5%) |
| 2 | BE | Sustained-проверка: дрейф держится → триггер |
| 3 | BE | Ждём market-open + свежую цену |
| 4 | BE→CT | Keeper: `reweight()` — продать дельту NVDA, докупить остальное (своп с AP) |
| 5 | CT | Веса вернулись к цели, value-preserving |
| 6 | FE | Дрейф ~0, запись в лог |

## 7. Безопасность / инварианты

- **Манипуляция оракулом на триггере** — главный риск; митигация: sustained-проверка (не мгновенный спайк), market-open.
- **Slippage cap** на свопе.
- **Издержки:** частый ребаланс = gas + проскальзывание (экономика fixed-weight, см. cap-vs-fixed-weight).
- **Redeem не паузится.**
